# OpenChapters — Chapter Template

This repository is a starting point for authors contributing a new chapter to the [OpenChapters](https://github.com/OpenChapters) project. It contains everything you need to write, typeset, and preview a single chapter as a standalone PDF.

## Prerequisites

You need a working **TeX Live** installation (2024 or later) which includes:

- **pdflatex** — the LaTeX compiler
- **arara** — the build automation tool (included in TeX Live)
- **biber** — the bibliography processor (included in TeX Live)
- **makeindex** — the index processor (included in TeX Live)

A full TeX Live installation is recommended to avoid missing-package errors.

**Supported platforms:** macOS, Windows, Linux

**Recommended editors:** TeXShop (macOS), TeXstudio (all platforms), VS Code + LaTeX Workshop, Overleaf (online)

## Quick Start

```bash
# 1. Clone this template
git clone https://github.com/OpenChapters/OCchaptertemplate.git MyChapter
cd MyChapter

# 2. Rename the chapter file
mv chapter/MyChapter.tex chapter/YourChapterName.tex

# 3. Update main.tex to point to your file (line 59):
#    \include{chapter/YourChapterName}

# 4. Edit chapter/YourChapterName.tex — fill in your content

# 5. Compile
arara -w main.tex
```

Your PDF will be generated as `main.pdf`.

## Step-by-Step Guide

### 1. Set Up Your Chapter

1. **Clone** this repository (or download as ZIP)
2. **Rename** `chapter/MyChapter.tex` to match your chapter name (e.g., `chapter/FluidDynamics.tex`)
3. **Edit `main.tex`** — change the `\include` line (near the bottom) to reference your new filename:
   ```latex
   \include{chapter/FluidDynamics}
   ```

### 2. Choose Your Chapter Abbreviation

Every chapter needs a **unique 6-character uppercase abbreviation** (called `\chabbr`). This prefix is used in all labels throughout your chapter. Examples from existing chapters:

| Chapter | Abbreviation |
|---|---|
| Linear Algebra | `LINALG` |
| Tensors | `TENSOR` |
| Number Systems | `NUMSYS` |
| Projections | `PROJTS` |

Edit the `\renewcommand{\chabbr}{MYCHAP}` line in your `.tex` file to set your abbreviation.

### 3. Edit the Chapter Header

At the top of your `.tex` file, fill in:

```latex
% Your name and affiliation:
\OCchapterauthor{Jane Smith, MIT}

% Your 6-character abbreviation:
\renewcommand{\chabbr}{FLUIDY}

% Chapter title:
\chapter{Introduction to Fluid Dynamics}\label{\chabbr:ch:FluidDynamics}

% Author registration (repeat for each co-author):
\writeauthor{\chabbr:ch:FluidDynamics}{Introduction to Fluid Dynamics}{Smith}{Jane}{Department of ME}{MIT}{jsmith@mit.edu}{https://jsmith.mit.edu}
```

### 4. Write Your Content

Use standard LaTeX for your content. The OpenChapters style provides:

- **Sections and subsections** with labels
- **Equations** (numbered, with labels)
- **Figures** (placed in `chapter/pdf/`)
- **Tables** (standard `tabular` environment)
- **Bibliography** (BibTeX entries in `chapter/chaptercitations.bib`)
- **Index entries** (using `\indexit{term}`)
- **Learning Objectives** (at the start of the chapter)

### 5. Label Convention

All labels **must** use the format `\chabbr:TYPE:name`:

| Type | Meaning | Example |
|---|---|---|
| `ch` | Chapter | `\label{\chabbr:ch:FluidDynamics}` |
| `sec` | Section | `\label{\chabbr:sec:introduction}` |
| `ssec` | Subsection | `\label{\chabbr:ssec:viscosity}` |
| `eq` | Equation | `\label{\chabbr:eq:navier_stokes}` |
| `fig` | Figure | `\label{\chabbr:fig:flow_diagram}` |
| `tb` | Table | `\label{\chabbr:tb:properties}` |

Reference other items with `\ref{\chabbr:sec:introduction}` or `Eq.~\ref{\chabbr:eq:navier_stokes}`.

### 6. Adding Figures

1. Create your figure as a **PDF** file (vector graphics preferred)
2. Place it in `chapter/pdf/`
3. Include it in your chapter:
   ```latex
   \begin{figure}[htbp]
       \centering
       \includegraphics[width=0.7\textwidth]{\graphicspath myplot.pdf}
       \caption{\label{\chabbr:fig:myplot}Description of the figure.}
   \end{figure}
   ```

Optionally, keep the original EPS source in `chapter/eps/` for archival.

### 7. Adding Citations

1. Add BibTeX entries to `chapter/chaptercitations.bib`
2. Cite them in your text with `\cite{key}` or `\textcite{key}`
3. The bibliography is automatically generated by biber

### 8. Compile Your Chapter

**Using arara (recommended):**
```bash
arara -w main.tex
```

**Using your editor:** Most LaTeX editors (TeXShop, TeXstudio, VS Code) will detect the `% !TEX TS-program = arara` directive and use arara automatically.

**Manual compilation (if arara is not available):**
```bash
pdflatex -shell-escape main
makeindex -s style/StyleInd.ist main
biber main
pdflatex -shell-escape main
pdflatex -shell-escape main
```

### 9. Update Metadata

Edit `chapter/chapter.json` with your chapter's metadata:

```json
{
  "title": "Introduction to Fluid Dynamics",
  "authors": ["Jane Smith"],
  "description": "An introduction to viscous and inviscid fluid flow.",
  "toc": ["Introduction", "Governing Equations", "Worked Examples"],
  "entry_file": "FluidDynamics.tex",
  "cover_image": "cover.png",
  "keywords": ["fluid dynamics", "Navier-Stokes", "viscosity"],
  "chapter_type": "topical",
  "chabbr": "FLUIDY",
  "depends_on": [],
  "discipline": "mse",
  "published": false
}
```

- **toc**: list your section titles in order
- **chapter_type**: `"topical"` or `"foundational"`
- **chabbr**: must match the `\chabbr` in your `.tex` file
- **depends_on**: list the `chabbr` codes of foundational chapters your chapter references (e.g., `["LINALG", "TENSOR"]`)
- **published**: leave as `false`; the editors will set this to `true` when accepted

## Referencing Other Chapters

If your chapter references equations, figures, or sections from other OpenChapters chapters, use their `\chabbr` prefix:

```latex
% Reference an equation from the Tensors chapter:
As shown in Eq.~\ref{TENSOR:eq:alphadef}, ...

% Reference a section from the Linear Algebra chapter:
See Section~\ref{LINALG:sec:eigen} for details on eigenvalues.
```

These cross-references will resolve when your chapter is compiled as part of the full OpenChapters book. In standalone mode, they will show as "??" — this is expected.

## Chapter Header Image

Each chapter has a banner image displayed at the top of its first page:

- **Size:** 2480 x 1240 pixels, 300 DPI, RGB
- **Format:** PDF
- **Filename:** `{CHABBR}header.pdf` (e.g., `FLUIDYheader.pdf`)
- **Location:** `chapter/pdf/`

If you don't provide a header image, the chapter will compile without one (using `\noheaderimage`). The editors can generate one for you.

To use your own header, replace `\noheaderimage` in your `.tex` file:
```latex
\chapterimage{FLUIDYheader.pdf}
```

## Submitting Your Chapter

When your chapter is ready for review:

1. Ensure your chapter compiles without errors
2. Fill in all `REPLACE:` placeholders
3. Complete the `chapter.json` metadata
4. **Option A:** Create a ZIP of your `chapter/` directory and email it to the editors
5. **Option B:** Fork the [OpenChapters](https://github.com/OpenChapters/OpenChapters) repository, add your chapter to `src/YourChapterName/`, and submit a pull request

The editors will:
- Review your content
- Generate a header image (if not provided)
- Assign a cover thumbnail
- Set `published: true` in `chapter.json`
- Merge your chapter into the OpenChapters catalog

## Troubleshooting

| Problem | Solution |
|---|---|
| `arara` not found | Ensure TeX Live is installed and `arara` is on your PATH. Try: `arara --version` |
| Missing package error | Install the full TeX Live distribution, or use `tlmgr install <package>` |
| Bibliography not appearing | Make sure you have entries in `chaptercitations.bib` and `\cite{}` commands in your text |
| Cross-references show "??" | This is normal for references to other OpenChapters chapters in standalone mode |
| Figures not found | Ensure PDF figures are in `chapter/pdf/` and the filename matches exactly |
| Index not building | Run `arara -w main.tex` (the `-w` flag is important for whole-file scanning) |
